# 奇货可查主页详细分析

## 一、整体设计风格

### 配色方案
- **主色调**：深蓝色（导航栏、按钮）
- **强调色**：
  - 红色：涨、看多、正向指标
  - 绿色：跌、看空、负向指标
  - 橙色：中性、警告
  - 蓝色：链接、操作按钮
- **背景色**：
  - 主背景：浅灰色/白色
  - 卡片背景：白色
  - 深色模式：支持切换

### 视觉特点
- **数据密集型**：大量数据表格、图表、数字
- **实时更新**：行情数据实时刷新
- **信息层级清晰**：通过颜色、字号、间距区分重要性
- **专业金融风格**：类似Bloomberg、Wind等专业金融终端

## 四、技术实现要点

### 前端技术栈（推测）
- **框架**：可能是Vue.js或React
- **图表库**：ECharts（大量使用）
- **UI组件**：
  - Tom Select（搜索框）
  - xm-select（多选框）
  - 自定义卡片组件
- **实时更新**：WebSocket或轮询
- **响应式设计**：适配桌面和移动端

### 数据特点
- **实时性**：行情数据实时刷新
- **大数据量**：需要分页、懒加载
- **多维度**：商品、席位、指数、基本面等多个维度

### 交互特点
- **快速访问**：顶部导航固定
- **信息密度高**：充分利用屏幕空间
- **可定制**：支持关注、收藏
- **权限控制**：VIP功能标识

## 六、用户体验设计

### 信息架构
- **核心指标前置**：头条精华在最显眼位置
- **个性化推荐**：只看关注、关注商品/席位
- **多层次导航**：顶部菜单+侧边栏+底部行情条

### 视觉层级
- **一级信息**：头条精华（大卡片、醒目颜色）
- **二级信息**：AI研报、7x24资讯（中等尺寸）
- **三级信息**：侧边栏内容（较小尺寸）

### 交互反馈
- **Hover效果**：链接、按钮有hover状态
- **加载状态**：数据加载时有loading提示
- **实时更新提示**：新数据到达时有视觉提示

## 七、响应式设计

### 桌面端（>1024px）
- 三栏布局：主内容区 + 右侧边栏
- 头条精华横向6个卡片

### 平板端（768px-1024px）
- 两栏布局：主内容区 + 右侧边栏（可能折叠）
- 头条精华可能换行

### 移动端（<768px）
- 单栏布局
- 侧边栏移至底部或抽屉
- 头条精华纵向堆叠

## 九、需要注意的细节

### 1. 颜色语义
- 红色=涨=看多=正向
- 绿色=跌=看空=负向
- 这与国际市场相反（国际是绿涨红跌）

### 2. 数据格式
- 价格：整数或小数点后1-2位
- 涨跌幅：百分比，带正负号
- 金额：亿元为单位

### 3. 时间格式
- 日期：12月24日、2025-12-23
- 时间：12/24 21:02

### 4. VIP标识
- 部分功能标注"VIP"
- 需要明确区分免费和付费功能

### 5. 免责声明
- "本数据仅供参考，不构成投资建议"
- 需要在适当位置显示
